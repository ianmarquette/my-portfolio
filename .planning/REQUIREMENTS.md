# Requirements: My Portfolio

**Defined:** 2026-03-13
**Core Value:** A recruiter or hiring manager can read a case study and clearly understand the designer's thinking process, not just the final output.

## v1 Requirements

### Home Page

- [ ] **HOME-01**: User sees a hero section with designer's name, role, and short tagline
- [ ] **HOME-02**: User sees a visual grid or list of all case studies with cover image and title
- [ ] **HOME-03**: User sees a brief about blurb on the home page
- [ ] **HOME-04**: User can find a contact link (email) on the home page

### Case Studies

- [ ] **CASE-01**: Each case study has a dedicated page with its own URL
- [ ] **CASE-02**: Each case study page opens with a cover image or hero visual
- [ ] **CASE-03**: Each case study page renders rich content sections (text + images alternating)
- [ ] **CASE-04**: Each case study covers problem/context, process, final designs, and outcomes
- [ ] **CASE-05**: User can navigate to next/previous case study from within a case study page
- [x] **CASE-06**: Case study content is authored in markdown files (no code changes to update content)

### Performance & Accessibility

- [ ] **PERF-01**: Pages load in under 2 seconds on a standard connection
- [ ] **RESP-01**: Layout is fully responsive across mobile, tablet, and desktop viewports

## v2 Requirements

### Future Enhancements

- **V2-01**: Dedicated About page with full bio
- **V2-02**: Resume/CV download link
- **V2-03**: Password-protected case studies for NDA work
- **V2-04**: Dark mode toggle

## Out of Scope

| Feature | Reason |
|---------|--------|
| CMS with database/admin UI | Markdown files sufficient for 3-5 case studies |
| Blog or writing section | Not the focus for this portfolio |
| OAuth or user accounts | Public portfolio, no auth needed |
| Heavy animations or interactions | Clean/minimal aesthetic — not the goal |
| Mobile app | Web-only |
