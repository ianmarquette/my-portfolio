# My Portfolio

## What This Is

A personal portfolio website for a product designer targeting full-time employment. It showcases 3-5 detailed case studies covering problem context, process, final designs, and measurable outcomes — giving hiring managers and recruiters the depth they need to evaluate fit.

## Core Value

A recruiter or hiring manager can read a case study and clearly understand the designer's thinking process, not just the final output.

### Active

- [ ] Home page with introduction and case study listing
- [ ] 3-5 individual case study pages (problem, process, visuals, outcomes)
- [ ] Clean, minimal aesthetic that keeps focus on the work
- [ ] Content editable via markdown files (no code changes needed to update)
- [ ] Contact / about section so recruiters can reach out
- [ ] Fast, responsive across devices

### Out of Scope

- CMS with database or admin UI — markdown files are sufficient for this volume
- Blog or writing section — not the focus
- Password-protected case studies — public portfolio for job search
- Animations or heavy interactivity — clean and minimal is the goal

## Context

- Targeting full-time product design roles; primary audience is recruiters and hiring managers
- 3-5 case studies, each long-form with: problem/context, process documentation, final design visuals, outcomes/metrics
- Content management via markdown files preferred — easy to update without touching code
- Visual direction: clean and minimal, whitespace-forward, restrained typography
- Success metric: designer feels proud and confident sending the link

## Constraints

- **Content**: Only 3-5 case studies — avoid over-engineering content infrastructure
- **Aesthetic**: Minimal — no heavy frameworks, flashy animations, or visual noise
